Phishing Detection Guide

01

Typosquatting

Attackers register domains that look like real ones — paypa1.com vs paypal.com. Look for number substitutions (0→o, 1→l) and extra words.

02

IP Address URLs

Legitimate sites use domain names — never raw IPs like http://192.168.1.1/login. An IP in a URL is almost always a red flag.

03

HTTP vs HTTPS

Phishing sites often use http:// (no padlock). While HTTPS doesn't guarantee safety, HTTP on a login page is always suspicious.

04

Suspicious TLDs

Domains ending in .xyz .tk .ml .gq .cf are free and commonly abused by phishers. Treat them with extra suspicion.

05

Keyword Stuffing

URLs like secure-paypal-verify-login.com cram in trust words to fool you. Real PayPal uses paypal.com — nothing more.

06

Long Subdomains

paypal.com.account-verify.malicious.net — the real domain is malicious.net. Read URLs right-to-left from the first single slash.
